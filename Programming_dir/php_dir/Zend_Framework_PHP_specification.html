Zend_Framework_PHP_specification Page

从宽约束

1.每行不超过130列

2.类名由英文组成，首字母大写，用下划线隔开，格式：This_Class_Do

3.接口类由Interface作后缀，如：Log_Adapter_Interface

4.文件命名，全部小写，多词用下划线隔开

5.由英文组成，不用数字和下划线，骆驼法则（首字母小写开始，其它首字母大写）

6.对象的存取器由get,set作为前缀

7.方法涉及到设计模式，则方法命名尽量包含设计模式名称

8.变量命名也遵循骆驼法则，不提倡数字

9.对于protected,private类成员变量，则以下划线开头

10.函数和变量保持一定程序的详细和冗长，要有实际意义，除了临时变量

11.常量使用字母、数字、下划线，且所有字母大写

12.不提倡全局函数和全局常量

13.纯字串赋值用单引号，如有单引号或变量则用双引号

14.注意留空格，等号，分号，连接号等要留空格

15.连接号对字串不留空格，对变量留空格

16.长行拆分，要用空格填充，使行具有连贯性和可读性，又不影响下一行的识别

17.如在数组中的各元素逗号后加空格增强可读性

18.对于数组长元素，各分行，各元素按列对齐

19.大括号在类名的下一行

20.每个类都有一个遵守PHPDocumentor标准的注释文档块

21.一个文件只允许一个类

类示例 

/**

*文档注释块

*/

class SampleClass

{

   //类的内部代码

   //必须缩进4个空格

}

22.类成员变量必须使用protected, private, public声明

23.类内部函数必须使用protected, private, public声明

24.函数大括号用法与类一致，但内部的大括号则直接跟在行尾

函数示例： 


/**

*文档注释块

* @param

* @return

*/

function sampleMethod($a)

{

   //函数的内部内容

   //必须缩进4个空格

}

25.函数的各参数逗号后加空格，增加可读性

26.if…else语句， 

if ($a != 0) {

   //内容

} else {

   //内容

}

27.switch结构 

switch ($numPeople) {

   case 1:

       break;

   case 2:

       break;

   default:

       break;

}

最后必须有一个default语句，不能省略

如需要省掉break或return与下一个case结合，那行在省掉的break处加注释

28.文件级文档块 

/*

*关于本文件的简要说明

*

*关于本文件的详细描述（如果有的话）...

*

* @copyright Shuky

* @version   CVS: $Id:$

* @since     File available since Release 1.2.0

* @date              2009-3-6

*/

29.类级别文档块 

/**

*类的简单说明

*

*类的详细说明(如果有的话)...

*

* @copyright Shuky

* @version   Release: @package_version@

* @since     Class available since Release 1.2.0

* @deprecated Class deprecated in Release 2.0.0

* @date             2009-3-6

*/
